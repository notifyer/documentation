Notifyer Documentation

Notifyer is a Web Push As a Service that you can easily integrate your web sites to send targeted push notifications to your customers. In this documentation, you can see several ways of Notifyer, customize notification flows in an easy way.

1. Overview

a. What is Web Push Notification?

Web push notifications are the messages that comes from specific site for better user re-engagement. Push notifications can be shown up even if your browser is closed, and any kind of user detail like email, user_id, etc. is not required.

Push Notifications requires HTTPS protocol in order to push / receive messages. You need to have at least one SSL endpoint to activate push notifications. We will add support for non-ssl domains later.

b. Browser Support

When we talk about stability, currently Firefox and Chrome browser supports web push notifications. When you look at supported versions of the browsers here, you will see that, you will be able to reach %55+ audience of global market when you use web push notifications. Currently, we are supporting chrome and firefox on desktop and mobile platform, however other browsers are also planning to integrate web push API to their browsers

If you prefer to not use the automatic generator, push a branch named gh-pages to your repository to create a page manually. In addition to supporting regular HTML content, GitHub Pages support Jekyll, a simple, blog aware static site generator. Jekyll makes it easy to create site-wide headers and footers without having to copy them across every page. It also offers intelligent blog support and other advanced templating features.
